# 技术概述
SUBFROST 作为比特币的去中心化托管方运作：支撑其合成资产的 BTC 由一组分布式签名者共同保障，而非由任何单一方掌控。
其核心采用灵活轮次优化 Schnorr 门限签名（FROST）与鲁棒异步 Schnorr 门限签名（ROAST）方案，允许多达 255 个参与者组成的签名组集体签署交易。

这种去中心化的托管方式由基于 QUIC 和 libp2p 构建的点对点网络管理，确保通信稳健且安全。

在接下来的页面中，我们将介绍 SUBFROST 的核心组件，包括但不限于：

- [**FROST 与 ROAST 密码学**](/key-components/frost-and-roast)：驱动去中心化托管的门限签名方案。
- [**Alkanes 协议**](/key-components/alkanes)：将 SUBFROST 锚定到比特币区块链的元协议。
- [**点对点网络**](/subfrost-networking/introduction-to-subp2p)：基于 QUIC 和 libp2p 构建的通信层。
