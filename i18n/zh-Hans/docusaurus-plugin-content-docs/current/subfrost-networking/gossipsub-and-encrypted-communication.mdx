# Gossipsub 与加密通信

SUBFROST 使用 [`gossipsub`](https://github.com/libp2p/specs/tree/master/pubsub/gossipsub) 协议，在节点之间实现高效且可扩展的实时消息传递。`gossipsub` 是一种发布/订阅（pub/sub）协议，节点可以订阅主题并接收发布到这些主题的消息。

它是 `subp2p` 网络层的核心组件，被用于多种任务，例如广播交易和区块信息，以及协调 FROST 签名仪式等多方计算。

## 主题（Topics）

`gossipsub` 主题是简单的字符串，用于标识特定的消息通道。节点可以订阅任意主题，并会收到其所连接的节点发布到该主题的所有消息。

在 SUBFROST 中，主题被用来为不同用途创建逻辑通信通道。例如，可能存在一个主题用于某个特定 `frBTC` 合约相关的所有消息，也可能存在一个主题对应某个特定的 FROST 签名组。

## 加密通信通道

尽管 `subp2p` 网络上的所有通信在传输层都是加密的（握手使用 `noise` 协议），但 `gossipsub` 本身并不为消息提供端到端加密。这意味着任何订阅了某个主题的节点都可以读取发布到该主题的所有消息。

然而，可以在 `gossipsub` 之上构建一层端到端加密，从而创建私密的加密通信通道。这是一种强大的模式，可用于构建安全的群聊、私密数据共享，以及其他需要保密性的应用。

### 一种安全群组通信的模式

在 `gossipsub` 之上创建加密通信通道的常见模式如下：

1.  **建立共享密钥：** 一组希望私密通信的节点，首先必须就一个共享密钥达成一致。这可以通过 Diffie-Hellman 之类的密钥交换协议完成，也可以从一个共同的密钥（例如 FROST 群组公钥）派生得到。

2.  **派生主题和加密密钥：** 随后可以使用该共享密钥，同时派生出一个唯一的 `gossipsub` 主题和一个对称加密密钥。例如，可以将共享密钥的哈希值用作主题，而将密钥本身用作加密密钥。

3.  **加密并发布：** 当某个节点想要向群组发送消息时，它会先使用共享加密密钥对消息进行加密，然后将加密后的消息发布到派生出的 `gossipsub` 主题。

4.  **接收并解密：** 只有属于该群组、且持有该共享密钥的节点才能派生出对应主题、订阅该主题并解密这些消息。

这种模式在公开的 `gossipsub` 网络之上有效地创建了一条私密的端到端加密通信通道。它有力地展示了 SUBFROST 的模块化组件如何组合起来，构建出复杂且安全的去中心化应用。

## SUBFROST 中的 `gossipsub`

`subfrost-cli` 使用 `gossipsub` 向网络广播消息，并参与 FROST 签名仪式。当你运行 `subfrost-cli` 时，它会订阅一组用于常规网络通信的默认主题，同时也会订阅与其所属 FROST 签名组相关的特定主题。

### 发布消息与订阅主题

以下是使用 `gossipsub` 发布消息和订阅主题的示例：

```rust
use subp2p::Subp2p;
use subp2p::gossipsub::{GossipsubEvent, IdentTopic as Topic};

// Create a new subp2p node
let mut subp2p = Subp2p::new().await?;

// Subscribe to a topic
let topic = Topic::new("my-topic");
subp2p.subscribe(&topic)?;

// Publish a message to the topic
subp2p.publish(topic.clone(), "my-message".as_bytes())?;

// Wait for a message to be received
loop {
    if let Subp2pEvent::Gossipsub(GossipsubEvent::Message { propagation_source, message_id, message }) = subp2p.next_event().await {
        println!("Received message: {:?}", message.data);
        break;
    }
}
```
