# 权益证明

SUBFROST 正在朝着一种由**经济权益**来保障托管 BTC 的网络安全的模型迈进。本页将坦诚地描述这个构想，以及它目前所处的阶段。当前的托管运行在早期的许可制阶段（参见 [frBTC 锚定与托管](./frbtc-peg-and-custody)）；开放的、基于质押的参与方式仍在规划中，尚未达到实际运行的规模。

## 为什么需要质押

要让托管真正实现无许可化，任何人都应该能够帮助持有支撑 frBTC 的密钥。在没有中心化守门人的情况下，经济权益正是保持这一切诚实运转的关键：

- 要加入签名群体，参与者需要锁定一定的价值。
- 他们在群体中的权重与所锁定的数量成正比，因此影响力与切身利益挂钩。
- 不当行为可能导致这部分权益被没收，这使得攻击协议的代价变得高昂。

这使得那些保障锚定安全的人，其利益与协议的长期健康保持一致，而不是依赖一个固定的、被信任的集合。

## 签名者做什么

签名者是参与门限签名的参与者（参见[签名与密钥](./signing-and-keys)）。他们的职责是协助授权提款：当有人销毁 frBTC 以赎回 BTC 时，必须有达到门限数量的签名者共同协作才能为这笔付款签名。一个更庞大、激励机制更完善的签名者集合，会让托管更难被攻陷。

## 奖励与惩罚

预期的经济机制是直截了当的：

- **奖励。** 签名者因可靠地参与而获得协议费用的一部分分成。
- **惩罚。** 做出损害协议行为的签名者可能会失去部分质押的权益，从而对攻击行为形成威慑。

## 目前的进展

:::info[发布前确认质押资产与具体机制]
如今的托管是许可制的：一个固定的签名者集合加上法定人数机制，而非开放式质押。本页所描述的开放式、按权益加权的模型是规划中的目标（Phase 2），其参数尚未确定。旧文档描述的是质押 `FUEL` 加 `frBTC`、一个流动性质押凭证代币，以及惩罚（slashing）机制，但目前生产环境中的质押是基于 DIESEL 的。发布前请对照合约确认真实的资产、权重如何计算、是否存在流动性质押凭证与惩罚机制，以及具体的推出计划。来源：`Diagnostico-docs.md` §10 C、`p-tokens__frBTC-roadmap.md`。
:::
