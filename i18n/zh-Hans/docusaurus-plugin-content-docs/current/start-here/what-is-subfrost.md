# 什么是 SUBFROST

SUBFROST 是一种使用你的 Bitcoin 参与去中心化金融（DeFi）的方式，既不需要把它交给某家公司，也不需要离开 Bitcoin 区块链。

如今，如果你想交易你的 Bitcoin、赚取收益，或者像人们在 Ethereum 或 Solana 上使用资金那样使用它，通常都得把它发送到别处。这意味着要么送去中心化交易所，要么通过由单一托管方运营的"wrapped BTC"跨链桥送到另一条链上。这两种情况下，你都在信任某一方不会丢失、冻结或滥用你的币。

## 问题所在

Bitcoin 是加密货币领域规模最大、最安全的网络。但它的基础层从一开始就不是为运行应用而设计的。原生 BTC 无法直接接入人们在 DeFi 其他领域使用的工具：自动做市商（用于兑换）、借贷市场，或隐私工具。

因此，任何想要不止是持有 Bitcoin 的人，都面临两个不理想的选择：

1. **把它发送到中心化交易所。** 现在是一家公司持有你的币。
2. **将它跨链桥接为"wrapped BTC"。** 现在是一个桥运营方（往往是一个小型托管方）持有真正的 Bitcoin，而你持有的是另一条网络上的一张欠条（IOU）。

这两个选择都意味着放弃托管权，并信任他人。**如果没有一个信任最小化的托管方，"Bitcoin DeFi"最终会依赖那些依赖信任的跨链替代方案，而这根本算不上真正的 Bitcoin DeFi。**

## 核心理念

一组独立的签名者共同持有支撑该系统的 Bitcoin。他们使用一种叫做**阈值签名（threshold signing）**的方法，控制资金的密钥被拆分到整个群体中。没有任何单一签名者持有完整密钥，因此没有任何单一方可以独自转移资金。即便某个签名者被攻破，资金依然安全。

用一句话概括：**SUBFROST 是 Bitcoin 的去中心化托管方。** 它作为一个轻量的 layer-0 运行，监视 Bitcoin 区块链，并协调一圈签名者，由他们锁定原生 BTC，并在你需要时将其释放。

## 你该如何实际使用它

当你将 Bitcoin 存入 SUBFROST 时，你会收到 **frBTC**，一种由你的 BTC 按 1:1 支撑、并原生结算在 Bitcoin 上的代币。最棒的是什么？这一切可以在同一笔交易中原子化完成，与其他操作一起进行。就这样，用户第一次可以直接用原生 BTC 兑换 DIESEL 或 FIRE 这样的 Bitcoin 代币，而完全不必知道自己其实与 frBTC 合约进行了交互。

- 将你的 BTC **包装（Wrapping）**为 frBTC 是即时完成的。
- 将 frBTC **解包（Unwrapping）**回原生 BTC，需要由分布式的签名者群体授权，而非任何单一个人。

有了 frBTC 在手，你就可以进行诸如兑换成其他 Bitcoin 资产、向 AMM 提供流动性、赚取收益等操作，而这一切的数据始终不会离开 Bitcoin L1。数据不仅仅是结算在基础层上，它根本从未离开过。

## 它有何不同

- **你留在 Bitcoin 上。** SUBFROST 不是一条独立的区块链，也不是侧链。一切始终留在 Bitcoin L1 上，你永远不会失去对那些 sats 的可见性。
- **没有单一托管方。** 支撑 frBTC 的 Bitcoin 由数量不断增长的签名者共同持有，而不是由某一家公司持有。目前有 9 名签名者，具备扩展到 255 名的能力。
- **保留你的敞口。** 这个理念本质上很简单：永远不要为了使用你的 Bitcoin，而把它发送到另一条区块链上。

### 签名者集合的现状

托管目前由**九名签名者**持有，他们尚未公开身份。这是起点，而非终点。接下来规划了三个步骤：

1. 密钥按需迁移，使得**九名签名者中至少六名可公开识别身份（doxxed）**。
2. **签名者集合扩大**，超出今天的九人规模。
3. 签名者集合转型为一个**无许可质押网络**，最多可达 255 名签名者，任何人都无法阻止你参与其中。

所以，当你读到没有单一托管方持有你的 Bitcoin 时，这在今天是真实的。规模更大、更开放的网络仍在规划中，值得按它自己的时间表来评判。签名机制本身如何运作，见 [frBTC 锚定与托管](../protocol/frbtc-peg-and-custody)。
