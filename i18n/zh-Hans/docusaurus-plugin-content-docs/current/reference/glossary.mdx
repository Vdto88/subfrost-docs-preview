# 术语表

## 协议与代币

*   **SUBFROST：** 比特币的去中心化托管方。它将 BTC 转换为可编程的链上形式，并可换回，全程无需任何单一公司持有这些币。
*   **Alkanes：** 构建在比特币区块链之上的智能合约元协议。SUBFROST 即构建于其上。
*   **元协议（Metaprotocol）：** 构建在另一个协议之上的协议。Alkanes 就是比特币之上的一个元协议。
*   **Protorunes：** Alkanes 所兼容的标准。Alkanes 是与 protorunes 兼容的协议，而非 Runes 的分叉。
*   **Protostone：** 编码在比特币交易中的 Alkanes 执行单元。一笔交易可以携带多个 protostone（例如先转账，再铸造，然后调用）。
*   **Cellpack：** Alkanes 调用所携带的消息，是一个以目标合约和操作码开头、后跟输入参数的列表。
*   **Alkane ID：** 合约的地址，是标识其部署位置的 `[block, tx]` 数对。
*   **View function（视图函数）：** 一种针对索引器状态运行、不产生费用的只读调用，用于查询余额、模拟调用或读取合约的 ABI。
*   **Opcode（操作码）：** 合约上一个带编号的方法。调用会指定目标 Alkane ID 和一个操作码。
*   **Runestone：** 嵌入在比特币交易 `OP_RETURN` 输出中的消息。
*   **`OP_RETURN`：** 比特币脚本中的一个操作码，可在交易中嵌入少量数据。
*   **BRC2.0：** 比特币上一个兼容 EVM 的智能合约层（也称为 brc20-prog），运行状态源自比特币交易的 Solidity 合约。
*   **`frBTC`：** Alkanes 内部由比特币支撑的价值单位，与 BTC 保持 1:1 锚定，可在整个比特币 DeFi 中使用。
*   **DIESEL：** 协议的公平发行原生代币，其发行量与比特币矿工手续费挂钩。
*   **FIRE：** FIRE 金库背后的代币，与通过预言机对协议国库定价的债券机制相绑定。
*   **`dxBTC`：** 一种生息的合成比特币资产（规划中）。
*   **`FUEL`：** 协议代币。
*   **AMM：** 自动做市商（Automated Market Maker）。一种依靠公式而非订单簿为资产定价的去中心化交易所。
*   **Wrap / Unwrap（包装 / 解包）：** 包装会锁定 BTC 并按 1:1 铸造 frBTC；解包则会销毁 frBTC，并由签名者组释放对应的底层 BTC。
*   **Oracle（预言机）：** 一个通过共识将链外数据（主要是价格）带上链的去中心化群体，确保没有任何单一方能左右数值。
*   **Signer set（签名者组）：** 通过门限签名共同托管支撑 frBTC 的 BTC 的一组参与者。
*   **PSBT：** 部分签名比特币交易（Partially Signed Bitcoin Transaction）。钱包在授权交易时签署的格式。

## 密码学

*   **Schnorr signature（Schnorr 签名）：** 比特币在 Taproot（BIP-340）中采用的签名方案，基于 secp256k1 曲线。其线性特性使得多个部分签名可以合并为一个签名。
*   **FROST：** Flexible Round-Optimized Schnorr Threshold（灵活轮次优化 Schnorr 门限签名）。一种门限签名方案，允许一组动态签名者共同生成一个 Schnorr 签名，而无需任何一方持有完整的私钥。
*   **ROAST：** Robust Asynchronous Schnorr Threshold（鲁棒异步 Schnorr 门限签名）。对 FROST 的一种稳健、异步优化，即使部分参与者迟缓或离线，签名过程仍能继续进行。
*   **DKG：** 分布式密钥生成（Distributed Key Generation）。一种仪式，参与者共同生成一个群组公钥，而每个人仅持有私钥的一个秘密份额。
*   **Keystore（密钥库）：** 保存用户敏感加密材料的加密文件（或硬件支持的存储）。
*   **AES-256-GCM：** 一种现代认证加密算法，同时提供机密性和完整性保障。
*   **PBKDF2：** 一种密钥派生函数，通过拉伸口令使离线暴力猜测的成本变得高昂。
*   **ASCII armoring（ASCII 铠装）：** 将二进制数据编码为人类可读文本，以便于存储和传输。

## 索引

*   **metashrew：** 基础索引器，针对比特币区块数据运行 Alkanes 的 WASM，并响应视图函数调用。它是链上状态的真实来源。
*   **subshrew：** v10 索引器，在 TiKV 之上运行 metashrew 运行时，将视图层与索引过程解耦。
*   **rockshrew-mono：** 一种以单一二进制文件搭配本地数据库运行 metashrew 运行时的方式。
*   **TiKV：** 用作 v10 索引器共享状态后端的分布式键值存储。
*   **ESPO：** 一种次级数据索引器，用于提供基础索引器不计算的批量历史数据和聚合数据（如交易笔数、持有者列表）。
*   **WASM：** WebAssembly。一种面向基于堆栈的虚拟机的二进制指令格式，合约和索引器均使用它。
*   **`wasip2`：** 一种标准化接口，用于在不同系统间运行 WebAssembly 模块。

## 网络

*   **`subp2p`：** SUBFROST 生态系统的基础点对点网络层，构建于 libp2p 之上。
*   **`subrelay`：** 网络栈中的一个组件，作为 `subp2p` 的会合点、流量中继和名称注册器。
*   **`subproxy`：** 连接 `subp2p` 网络与传统互联网之间的桥梁。
*   **`subtun`：** 基于 `subp2p` 网络的安全点对点 VPN。
*   **Gossipsub：** 一种发布/订阅协议，节点可以订阅主题并接收发布到这些主题的消息。
*   **noise：** `subp2p` 在节点间加密握手所使用的协议。
*   **Microservices（微服务）：** 通过安全的 `subp2p` 网络进行通信的小型独立服务。
*   **Kademlia DHT：** 用于在公共互联网上发现节点的分布式哈希表。
*   **mDNS：** 多播 DNS（Multicast DNS）。让节点无需中央服务器即可在局域网中相互发现。
*   **NAT traversal（NAT 穿透）：** 让位于防火墙和 NAT 之后的节点得以互相连接的技术。
*   **AutoNAT：** 一种让节点发现自身公网 IP 地址的协议。
*   **Hole punching（打洞，DCUtR）：** 一种借助中继帮助，让两个位于 NAT 之后的节点建立直接连接的技术。
*   **Circuit relay（电路中继）：** 当无法建立直接连接时，通过中继节点转发流量。
*   **TUN device（TUN 设备）：** 一种虚拟网络接口，从用户空间程序而非物理硬件接收数据包。
*   **SOCKS5：** 一种用于通过代理服务器路由流量的代理协议。
*   **Reverse proxy（反向代理）：** 一种代表客户端从一个或多个服务器获取资源的代理服务器。
