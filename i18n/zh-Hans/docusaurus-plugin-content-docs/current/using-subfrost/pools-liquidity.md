# 资金池与流动性

SUBFROST 上的每一笔交换都是与**流动性池（liquidity pool）**进行交易：这是一个由任何人都可以贡献的、包含两种代币的共享储备。当你提供流动性时，你会向资金池中存入一对代币，并赚取该资金池产生的交换手续费的一部分。

## 工作原理

- 一个资金池持有两种资产（例如 DIESEL 和 frBTC）。价格会随着交易自动调整，因此你总能找到交易对手方。
- 当你添加流动性时，你会获得一个代表你在资金池中份额的 **LP 仓位（LP position）**。
- 每当发生交换，都会产生手续费并累积到资金池中，因此你的仓位价值会随时间增长。
- 你可以随时通过赎回 LP 仓位，取回你的流动性（以及累积的手续费）。

## 交换手续费，以及你能赚到什么

一笔交换默认支付 **1%** 的手续费，分为两部分：

| 份额 | 费率 | 去向 |
| --- | --- | --- |
| 流动性提供者 | 0.8% | 进入资金池，累积到你的仓位中 |
| 协议 | 0.2% | 归协议所有 |

因此，**作为流动性提供者，你可赚取经由你的资金池成交的交易量的 0.8%**，按你在其中所占份额的比例分配。

1% 是一个默认值，而非一条硬性规则：手续费是按资金池单独设置的，应用会在向你报价时实时读取该资金池的费率。某个特定资金池的配置可能与此不同，你看到的报价费率就是实际适用的费率。

:::info[确认 1%（0.8% LP / 0.2% 协议）是否为对外公布的预期费率]
这些是资金池合约中的默认值，应用会精确镜像这些数值。有两点需要确认：这是否是你希望写入文档的费率，以及是否有任何生产环境的资金池配置偏离了 1% 的默认值。另外要注意，旧版文档曾将"1%、0.8%、0.2%"错误地归到包装和解包上，这是不对的：包装和解包收取的是 0.1%。数字是对的，只是用错了产品。
:::

## 提供流动性

1. 打开 **Pools（资金池）** 页面，选择一个资金池（或在其中新建一个仓位）。
2. 按照资金池的比例存入两种代币。
3. 确认并签名。此时你已持有一个 LP 仓位，并开始赚取该资金池交换手续费的一部分。

## DIESEL / frBTC 资金池

**DIESEL / frBTC** 资金池是 SUBFROST 经济体系的核心：为该资金池提供流动性的用户还会获得 **[FIRE](./fire-vault)**（协议的奖励代币）奖励。质押与债券认购如何在此基础上运作，详见 FIRE Vault 指南。

:::info[确认应用中的存入与提取流程]
交换手续费现已在上文写明。这里仍待完善的是具体步骤：请确认应用中确切的存入与提取流程，以及 LP 仓位是否以用户需要了解的某种方式呈现。
:::
