# 借贷

借贷让你可以**出借代币赚取利息**，或**以抵押品借入代币**，直接在 Bitcoin L1 上完成。这是点对点模式，你自行设定条款，并与其他用户匹配成交。没有资金池，没有中间商，你的资金会一直留在你的钱包中，直到匹配发生。

## 借贷速览

| 属性 | 数值 |
| --- | --- |
| 模式 | 点对点，固定利率，固定期限 |
| 结算 | 单笔 Bitcoin 交易 |
| 发布报价 | 免费，链下，无网络费用 |
| 谁支付结算费用 | 吃单方（接受报价的一方） |
| 协议费用 | 每笔借贷 3,000 sats，在结算时支付 |
| 利息 | 创建时固定，根据 APR 和期限计算 |
| 期限 | 以区块数计量 |
| 清算 | 无。活跃借贷不存在追加保证金 |
| 托管 | 无。没有托管方，也没有管理员密钥 |

## 工作原理

借贷有两方，你可以扮演其中任意一方：

- **贷方**发布**借贷报价**："若你提供 Y 抵押品，我将以此利率出借 X 代币。"
- **借方**发布**借贷请求**："我想借入 X 代币，并将提供 Y 抵押品。"

发布报价的一方是**挂单方**，接受报价的一方是**吃单方**。当有人接受你报价的对手方时，借贷会在一笔链上交易中创建，并立即变为活跃状态。

### 发布报价免费且即时

当你发布报价时，你的钱包会**对其签名，但不会将任何内容发送到 Bitcoin 网络**。因此：

- **发布无需网络费用。**
- **没有链上足迹**，也无需等待出块。你的报价会立即出现在订单簿中。
- **你的资金会一直留在钱包中。** 已发布的报价是一条预签名指令，按照设计**无法单独广播**。只有当吃单方接受并加入其一方后，它才会成为一笔真正的交易。

这意味着**只有吃单方需要支付结算费用**。作为挂单方，发布、编辑和取消报价都不需要任何费用。

### 撮合成交后借贷在链上结算

当吃单方接受你的报价时，会广播一笔 Bitcoin 交易，该交易会：

1. 将借出的代币交给借方，
2. 锁定借方的抵押品，并
3. 创建这笔借贷。

一旦该交易确认，借贷即变为**活跃**状态，其期限的计时开始。如果其中任何一部分失败，双方都会在同一笔交易中拿回各自的代币。你不会陷入"借了一半"的状态。

关于预签名报价如何在吃单方注资之前保持无效的具体机制，参见 [借贷协议](../protocol/lending)。

## 一笔借贷的成本

| 费用 | 谁支付 | 何时 |
| --- | --- | --- |
| 协议费用，3,000 sats | 吃单方 | 结算时，内置于交易中 |
| Bitcoin 网络费用 | 吃单方 | 结算时，随你选择的费率变化 |
| 发布、编辑、取消报价 | 无人 | 免费，不广播任何内容 |

## 你需要支付的利息

利息在**创建时即固定**，不会随时间累积。在你同意任何条款之前，就已知道确切的还款金额：

```
interest  = loan amount × APR × (duration in blocks / 52,560)
repayment = loan amount + interest
```

`52,560` 是一年中 Bitcoin 区块的数量（每天 144 个 × 365 天）。

**示例。** 以 12.5% APR 借入 1 DIESEL，期限 1,008 个区块（约一周）：

```
interest  = 1 × 0.125 × (1,008 / 52,560) = 0.00239726 DIESEL
repayment = 1.00239726 DIESEL
```

由于这个数字直接来自条款本身，双方都可以提前计算出来，并且它永远不会改变。

:::warning[利息四舍五入为零的条款会被拒绝]
利息使用整数运算计算，并向下取整。如果你的借贷金额、APR 和期限小到使利息向下取整为**零**，合约会拒绝创建该笔借贷。如果借贷无法通过，请提高金额、APR 或期限。
:::

## 抵押品限制

每笔借贷都是超额抵押的：抵押品的价值必须高于所欠款项。应用会按每种抵押代币强制执行**最高贷款价值比（LTV）**限制，以美元计量：

| 抵押品 | 最高 LTV | 最低抵押品（相对于所欠款项） |
| --- | --- | --- |
| frBTC | 80% | 约 125% |
| DIESEL | 50% | 约 200% |
| 其他任意代币 | 30% | 约 333% |

关于这一限制的应用方式，有两点：

- **它是针对还款金额计量的，而不是借入金额。** 需要覆盖的是本金加利息。提高 APR 或期限会提高还款金额，进而提高所需的抵押品。
- **高于最低要求的抵押品始终是允许的，低于则会被拒绝。** 应用不会允许你发布低于最低要求的报价，提交报价时还会再次执行这项检查。

### 价格变动可能导致你的未成交报价被移除

报价创建时的抵押率处于其抵押品最高 LTV 之内或以下，但发布之后价格会持续变动。如果未成交报价上的抵押品价值下跌到**低于其所欠款项**，该报价就会被**从订单簿中移除**。

这一规则**仅适用于尚未被任何人接受的未成交报价**。**活跃借贷永远不会被触碰**：一旦借贷存在，其条款就是固定的，并严格按照约定执行。详见下文。

:::info[价格不可用时，限制将暂停]
LTV 检查需要两种代币的美元价格。如果其中任何一种代币没有应用可解析的价格，该检查就无法执行，应用会允许你设置任意数值，同时警告你需要自行承担风险。请认真对待这个警告：这意味着没有任何机制在替你检查抵押品。
:::

:::info[确认 LTV 上限数值与表单预填内容]
上文的最高 LTV 数值（frBTC 80%、DIESEL 50%、其他代币均为 30%）以及移除阈值（未成交报价所欠款项超过其抵押品价值后即被移除）均来自应用的借贷策略，该策略是在最初的借贷文档写成之后才加入的。此前的借贷文档页面完全没有提到任何 LTV 强制执行机制。有两点需要确认：这些是否是团队希望公开的数值；以及报价表单为抵押品数量预填的默认值，因为它默认使用 150% 的比例——这个比例满足 frBTC 的最低要求，却低于 DIESEL 的最低要求。
:::

## 选择你的抵押品

抵押品条款在**借贷创建时即固定**，并**严格按照约定执行**。除了上文提到的**抵押品最低要求**之外，**不存在追加保证金，也不存在自动清算**。这带来一个重要后果：

> **如果你是贷方，你必须能够接受最终收到的*正是*约定的抵押品，即便其价格已经下跌到低于你所出借金额的价值。** 如果借方违约且抵押品价值已下跌，认领该抵押品是你唯一的追索手段。协议不会为你补足差额。

**LTV 上限限制了你的下行风险，但并不能消除它。** 它只在报价创建时按当下价格检查一次，一旦借贷变为活跃状态，就不会再有任何重新检查。因此，你应该要求一个即便最终拿到的是抵押品而非还款，你也**真正能够接受**的水平：

- **更高的抵押品**能在价格向不利方向变动时更好地保护你，但会让你的报价**吸引力降低**，借方可能永远不会接受。
- **更低的抵押品**更容易成交，对借方也更友好，但如果抵押品价值下跌，你将**承担更多风险**。

借方面临的是镜像情况：你提供的抵押品越多，如果无法还款，你所承担的风险就越大；但要求较少抵押品的报价可能很稀缺。

## 如何放贷

1. 打开**放贷 / 借款（Lend / Borrow）**页面，选择你想要**出借**的代币，以及你将接受作为**抵押品**的代币。
2. 将表单切换到**放贷（Lend）**。
3. 输入出借金额、你要求的抵押品数量、**APR**，以及以区块计的**期限**。
4. 点击**放贷（Lend）**，并在钱包中批准签名。

你的报价现在已在订单簿中生效。无需费用，也无需广播。

## 如何借款

1. 选择你想要**借入**的代币，以及你将作为**抵押品**发布的代币。
2. 在订单簿中浏览你中意的报价（按 APY 排序），**或者**将表单切换到**借款（Borrow）**并发布你自己的借贷请求。
3. 若要接受现有报价，点击**借款（Borrow）**。你会看到一份摘要：你需要提供什么、你将收到什么、APY、总还款金额，以及截止时间。
4. 确认并支付网络费用以完成结算。

## 一笔借贷的生命周期

一旦借贷变为活跃状态，可能发生以下情况。

### 按时还款，正常路径

- 借方需要在截止时间前偿还**本金加利息**。点击一次**还款并取回（Repay & reclaim）**，即可在同一笔交易中偿还借贷并取回抵押品。
- 随后贷方**认领还款**。
- 借贷完全结清。双方都得到了预期的结果。

### 违约，借方未能按时还款

- 如果截止时间已过而未还款，该借贷即被判定为**违约**。
- 贷方可以**认领抵押品**。一键操作即可同时完成违约判定和认领。
- 借方保留其借入的代币，但**丧失抵押品**。

借贷一旦违约，就无法再被偿还。截止时间一过，贷方唯一的路径就是认领抵押品。

:::info[截止时间一过，应用就会将借贷显示为已违约]
在链上，只有当有人实际触发时，借贷才会变为 `DEFAULTED` 状态。应用的判定更严格：只要截止时间落后于当前区块，它就会立即将逾期借贷显示为**已违约**，并撤下借方的还款按钮。不要指望截止时间与有人触发违约之间存在宽限期。
:::

你随时可以在借贷页面底部查看你的借贷及其状态（**活跃借贷（Active Loans）**、**已关闭借贷（Closed Loans）**，以及你的未成交**我的报价（My Offers）**）。

## 管理你的报价

你的未成交报价会出现在**我的报价（My Offers）**下。在那里你可以：

- **编辑（Edit）**一笔报价。这会重新签署一份新报价并取消旧的报价。
- **删除（Delete）**一笔报价，将其从订单簿中移除。

这两项操作都和发布一样，免费且在链下完成。

## 小贴士

- **从小额开始**，先熟悉整个流程，再发布较大的报价。
- **放贷前先考虑违约情形。** 为最终以抵押品而非你出借的代币获得偿付的情形，设定抵押品价值和利率。
- **合理设定你要求的抵押品数量。** 要求过高没人会接受，过低你会承担更多风险。**抵押品最低要求是一道底线**，而不是推荐值。
- 如果你是借方，**留意截止时间**。没有宽限期，一旦过期你就无法再还款。
