# FIRE Vault

**FIRE** 是 SUBFROST 的奖励代币。它奖励为 [DIESEL / frBTC 资金池](./pools-liquidity)提供流动性的用户，获取方式有两种：**质押（staking）**你的流动性，或**债券认购（bonding）**。

没有预挖（premine）。所有存在的 FIRE 都是由协议发行的，FIRE 退出流通的唯一方式是向国库赎回。

## FIRE 概览

| 指标 | 数值 |
| --- | --- |
| 最大供应量 | 2,100,000 FIRE |
| 小数位数 | 8（与 Bitcoin 相同） |
| 预挖 | 0% |
| 质押池 | 85%（1,785,000 FIRE） |
| 债券池 | 15%（315,000 FIRE） |
| 减半间隔 | 105,000 个区块（约 2 年） |
| 激活区块 | 950,420 |
| 首次减半 | 区块 1,050,000 |

FIRE 的发行量每 105,000 个区块减半一次，这是 Bitcoin 减半间隔的一半，因此每两次 FIRE 减半中就有一次恰好落在 Bitcoin 减半上。

## 金库如何向你支付

金库接受 **DIESEL / frBTC LP 代币**。它通过奖励累加器（标准的 Synthetix 质押模型）发放 FIRE：每个区块，固定数量的 FIRE 会按比例分配给所有加权质押份额。你在某个区块发行量中所占的份额，等于你的加权质押量除以金库中的加权质押总量，因此奖励是持续累积的，而不是离散发放的。

第 0 个纪元（epoch 0）的每区块发行量：

| 资金池 | 每区块基础单位 | 每区块 FIRE | 每日 | 每年 |
| --- | --- | --- | --- | --- |
| 质押（85%） | 850,000,000 | 8.5 | ~1,224 FIRE | ~446,250 FIRE |
| 债券（15%） | 150,000,000 | 1.5 | ~216 FIRE | ~78,750 FIRE |
| **总计** | **1,000,000,000** | **10** | **~1,440 FIRE** | **~525,000 FIRE** |

每日与每年的数字基于 Bitcoin 平均每天 144 个区块、每年 52,500 个区块计算。

## 发行时间表

每经过 105,000 个区块，每区块发行速率就会减半。纪元（epoch）边界对齐到以区块 0 为锚点的全局网格，而不是以激活区块为锚点，且每两次 FIRE 减半中就有一次与 Bitcoin 减半重合。质押池与债券池之间 85 / 15 的分配比例在每个纪元中都保持不变。

| 纪元 | 长度 | 每日总量 | 纪元结束时的累计供应量 |
| --- | --- | --- | --- |
| 0 | 105,000 个区块 | ~1,440 FIRE | 1,050,000 FIRE |
| 1 | 105,000 个区块 | ~720 FIRE | 1,575,000 FIRE |
| 2 | 105,000 个区块 | ~360 FIRE | 1,837,500 FIRE |
| 3 | 105,000 个区块 | ~180 FIRE | 1,968,750 FIRE |
| ... | ... | ... | 2,100,000 FIRE（上限） |

### 为什么这个上限不需要预挖

每个纪元的发行量恰好是前一个纪元的一半，因此总发行量是一个无穷等比级数（geometric series）。它收敛于供应上限：

```
total = epoch_0_emission x sum(1/2^i) for i = 0..infinity
      = epoch_0_emission x 2
      = 1,050,000 x 2
      = 2,100,000 FIRE
```

时间表本身就保证了这个上限。创世时不需要预留任何数量来让这些数字成立，这就是为什么预挖是 0% 的原因。

有一点值得说清楚：减半级数只会不断逼近其总和，却永远不会真正到达。供应量会无限接近 2,100,000 FIRE，但始终略低于这个数字，就像 Bitcoin 的 2,100 万上限本身是一个永远不会被真正铸造出来的天花板一样。这个上限是一个极限值，而不是一个会被达成的里程碑。

:::info[纪元 0 应该以区块 1,050,000 还是 1,055,420 为准？]
两份内部资料的说法并不一致。书面时间表称纪元边界遵循以区块 0 为锚点的固定网格，这会把首次减半定在区块 **1,050,000**，并使第一个纪元比其余纪元短（99,580 个区块，而不是 105,000 个），因为 FIRE 是在网格槽位进行到一半时的区块 950,420 激活的。而部署记录则将纪元 0 记为覆盖 **950,420 到 1,055,420**，正好是完整的 105,000 个区块，首次减半会因此晚 5,420 个区块。应用内的一条说明称，在该部署记录写下几天之后从合约本身读取到的数值是 1,049,999，这一点更支持第一种说法。这不只是细节问题：如果第一个纪元确实被截断，它发行的将是约 995,800 FIRE，而不是 1,050,000，整个序列最终会收敛于约 2,045,800，而不是 2,100,000。请确认哪一个数值才是线上实际生效的，以及第一个纪元是否本就应该被截断。
:::

## 锁仓倍数

你可以将质押锁定更长时间以获得更多收益。锁仓时间越长，同样数量的 LP 就能获得越高的奖励倍数。锁仓期不能超过当前纪元的到期时间。

| 锁仓期 | 区块数 | 奖励倍数 |
| --- | --- | --- |
| 不锁仓 | 0 | 1.0x |
| 1 周 | 1,050 | 1.25x |
| 1 个月 | 4,375 | 1.5x |
| 3 个月 | 13,125 | 2.0x |
| 6 个月 | 26,250 | 2.5x |
| 1 年 | 52,500 | 3.0x |

承诺锁仓的时间越长，你在质押发行中获得的份额就越大。

:::info[只有长期锁仓才会影响债券价格]
债券价格预言机（oracle）**只统计锁仓 6 个月或以上的 LP**，并且统计的是原始数量（raw amount），而不是乘以倍数后的加权值。短期质押者仍然可以获得 FIRE，但不会影响预言机。债券认购者需要支付的价格，是由那些长期承诺的人所决定的。
:::

## 质押你的 LP

1. **获取 DIESEL / frBTC LP。** 先向 [DIESEL / frBTC 资金池](./pools-liquidity)提供流动性。金库只接受这一种 LP 代币，其他一律不接受。
2. **选择锁仓时长。** 从不锁仓到锁仓一年皆可。更高的倍数会让同样数量的 LP 按比例获得更多 FIRE，但你的仓位在到期前将被锁定。
3. **质押并获得一个 NFT 仓位。** 这个 NFT 同时承载两样东西：你的 **FIRE 收益权**（可领取的发行量）和你的 **LP 索取权**（到期时可取回的流动性）。
4. **每个区块都在赚取收益。** 奖励按区块累积。你可以随时领取或复投。

```
Stake(LP, lock, amount) -> NFT position (FIRE yield + LP claim)
```

## 取回你的 LP

根据你想要的灵活度不同，共有三条退出路径。

### 解除质押（Unstake）

等待锁仓到期后，解除质押该 NFT。你将取回 LP，外加所有已累积的 FIRE。

```
Unstake(NFT) -> LP + FIRE
```

一旦锁仓到期，领取奖励这个操作本身就会为你完成取回：对于一个尚未拆分的仓位，到期后的一次领取会同时返还 FIRE **和** LP，并关闭该仓位。你不需要再单独执行解除质押。

### 拆分为 wLP

将 NFT 仓位拆分为两部分。**NFT** 保留 FIRE 收益权。一个可互换的 **wLP** 代币则承载 LP 索取权，你可以单独转让或出售它。合并（merge）操作可以将仓位重新组合到一起。

```
Split(NFT)       -> NFT (FIRE yield only) + wLP (fungible LP claim)
Merge(NFT + wLP) -> NFT (full position restored)
```

这就是在不放弃 FIRE 收益流的前提下，将 LP 流动性转移出去的方法。

### 赎回已到期的 wLP

一旦纪元到期，任何持有 wLP 的人都可以直接将其赎回为 LP，无需持有 NFT。

```
RedeemExpired(wLP) -> LP
```

## 债券认购（Bonding）

债券认购适合那些想要**立即**获得 FIRE，而不是随时间慢慢赚取的人。你将 LP 代币永久交给协议，作为交换，你会以相对于质押预言机价格 **10% 的折扣**获得 FIRE，并在约 **7 天**（1,050 个区块）内归属（vest）。该折扣是协议可以调整的一项设置，因此请将 10% 视为当前费率，而不是固定规则。

定价方式完全透明：

```
oracle_price    = max(LP_locked_6mo_or_more, 1 LP) x 10^8 / annual_FIRE_emission

effective_price = oracle_price x (10000 - discount_bps) / 10000

bond_price      = max(effective_price, floor_price)

FIRE_out        = LP_in x 10^8 / bond_price
```

预言机价格公式中的 `max(..., 1 LP)` 是一个启动保护（bootstrap guard）：在还没有任何人锁仓 6 个月或以上的 LP 之前，它能让价格保持有定义，而不会出现除以零的情况。

用于认购债券的 LP 会直接进入国库并留在那里。它不会被退还，并且会永久性地为赎回地板价提供支撑。

用大白话说，债券认购就是这样一笔交易：你把 LP 永久交给协议，换来的是以相对于年化质押收益 10% 的折扣获得的 FIRE，并在 7 天内归属。

### 地板价保护机制（floor guard）

注意 `bond_price` 公式中的 `max()`。这就是地板价保护机制，它的作用是阻止一次「免费的来回套利」。

一旦国库持有 LP 且 FIRE 已被铸造，地板价就是真金白银。如果没有这个保护机制，你就可以在低于地板价的价格认购债券，然后立即赎回，换取比你投入更多的 LP。有了 `bond_price = max(effective_price, floor_price)` 这条规则，认购债券的价格就永远不会低于赎回的价格：

```
Bond at floor_price    -> mint X FIRE  (LP goes into treasury)
Redeem X FIRE at floor -> burn for LP  (LP comes out of treasury)
                       = same LP back  (zero net change)
```

一个认购债券后立即赎回的人，净收益为零。7 天的归属期意味着债券认购者是在押注价格上涨，而不是从协议中抽取价值。

### 债券认购额度会随时间逐步累积

债券认购的额度是受限的，而不是无限的。额度从合约上线的那一刻起逐区块累积，因此刚上线不久时额度还很少，一笔较大的债券认购可能无法通过。如果一笔债券认购因额度不足被拒绝，这并不代表出了问题：等待一段时间再试，或认购更小的金额即可。

## 赎回与价格地板

任何 FIRE 持有者都可以销毁 FIRE，按比例换取国库中作为支撑的 LP：

```
floor_price = total_treasury_LP / total_FIRE_supply
```

**赎回需要收取 1% 的手续费**，这笔费用不会支付给任何人：它会留在国库中。因此你实际拿到的大约是你按比例应得份额的 99%，而你留下的那部分则会为所有仍持有 FIRE 的人略微推高地板价。

这是一个硬性地板价，并且会随着国库规模而变化。每一次债券认购都会存入永不离开的 LP，因此随着债券认购活动的增长，地板价长期来看只会朝一个方向移动。

### FIRE 没有市场可供套利

FIRE **没有 AMM 资金池，没有交易所上市，也没有兑换路径**。你无法像交易其他 Bitcoin 资产那样买卖它。获取 FIRE 的唯一方式是**质押** LP 来赚取它，或**债券认购** LP 以折扣价换取它。处置 FIRE 的唯一方式是向国库**赎回**它。

这就是为什么下面这两种价格不是市场报价。它们都是协议计算出的数值，FIRE 的「价格」指的就是其中之一。

如果平台之外确实出现了某个价格，同样的力量仍然会约束它：低于地板价时，任何人都可以买入并向国库赎回 LP 获利，从而把价格推回去；高于年化质押收益时，质押并卖出会成为更好的选择，从而吸收多余的需求。地板价是一个硬性下限，质押收益则是一个软性上限。

## 两种参考价格

除了市场报价之外，FIRE 还有两种不同的「价格」，它们回答的是不同的问题。

| 价格 | 公式 | 含义 |
| --- | --- | --- |
| **预言机价格（Oracle）** | `locked_LP / annual_emission` | 一个 6 个月质押者一年能赚到多少 |
| **地板价（Floor）** | `treasury_LP / total_supply` | 一个赎回者销毁 FIRE 能换回多少 |

如果你在任何地方看到「折扣」这个说法，请确认它是相对于这两者中的哪一个而言的。预言机价格和地板价回答的是不同的问题，相对于其中一个的折扣，并不等同于相对于另一个的折扣。

## 飞轮效应

质押、债券认购与赎回并非三个各自独立的功能，它们在一个闭环中相互驱动。

| 角色 | 行为 | 引发的结果 |
| --- | --- | --- |
| LP 提供者 | 质押 DIESEL / frBTC LP，锁仓越久获得的 FIRE 越多 | 长期锁仓会推高预言机价格 |
| 债券认购者 | 按预言机价格支付 LP，换取折扣价的 FIRE | 预言机价格越高，认购者每份 FIRE 需支付的 LP 就越多，国库因此更快充实 |
| 国库 | 永久保留每一笔债券认购存入的 LP | 国库规模越大，地板价就越高 |
| 赎回者 | 销毁 FIRE，按比例换取国库中的 LP | 地板价保护机制随之生效，使低价认购债券变得更难 |

可以把它理解为一个循环：更多长期质押者推高预言机价格，更高的预言机价格让认购者支付更多 LP，更多的 LP 推高地板价，而更高的地板价既保护了 FIRE 持有者，也限制了下一个认购者能以多低的价格买入。长期承诺的流动性推高地板价，地板价保护持有者，而发行量则始终由真实的长期承诺来调节。

## 风险

| 风险 | 对你意味着什么 |
| --- | --- |
| **启动期（Bootstrap）** | 早期锁仓的 LP 很少，因此预言机价格偏低，FIRE 会非常便宜，直到长期承诺增长起来 |
| **国库初始为空** | 地板价从 0 开始，只有当认购者存入 LP 后才会上升 |
| **锁仓流动性受限** | 锁仓的 LP 在到期前无法取出。如果需要部分灵活性，可以拆分为 wLP |
| **区块时间存在波动** | 这里的所有时长都假设 Bitcoin 平均约 10 分钟一个区块，实际经过的时间会有偏差 |

## 小贴士

- **让锁仓期匹配你的信心。** 锁仓越久按比例赚得越多，但你无法提前解除。
- **需要拿回流动性时使用拆分（Split）。** 把 FIRE 收益权留在 NFT 上，单独交易 wLP。
- **债券认购不是质押。** 债券认购让你立即获得 FIRE，但你的 LP 永久失去了。质押则会把 LP 连同奖励一起还给你。
- **留意减半。** 发行量每 105,000 个区块减半一次，因此早期纪元的收益最高。
- **国库增长即地板价增长。** 通过债券认购流入的 LP 越多，赎回地板价就越牢固。
